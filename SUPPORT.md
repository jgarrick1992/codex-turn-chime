# Support

Use the GitHub issue forms for reproducible bugs, feature proposals, status-detection problems, and platform compatibility reports. Include the application version, operating system, event source, and diagnostic results.

Before opening an issue:

1. Read [Troubleshooting](docs/troubleshooting.md).
2. Open **Diagnostics** and report pass/fail states without copying private local paths unless necessary.
3. Remove prompts, answers, commands, tokens, usernames, and transcript content.

Security vulnerabilities belong in private reporting described by [SECURITY.md](SECURITY.md). The project is community-maintained and does not provide a guaranteed response time or commercial support.
