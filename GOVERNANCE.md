# Governance

CodexTurnChime uses a maintainer-led, contribution-friendly model.

- Maintainers set scope, merge pull requests, publish releases, and enforce project policies.
- Decisions prioritize privacy, deterministic status semantics, platform safety, and maintainability.
- Material changes to the event schema, Hook behavior, data collection, network access, or platform scope require a public design issue before implementation.
- Contributor access is earned through consistent, constructive work and can be removed for inactivity or policy violations.
- Releases require CI success, release-asset review, checksums, SBOM, provenance, and a maintainer's manual publish action.

The repository owner is the initial maintainer. Additional maintainers will be listed here after they accept the responsibilities above.
